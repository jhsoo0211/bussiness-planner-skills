# 토큰 소모 가늠표 · 모델 추천

> 이 표의 수치·등급은 모두 **대략의 추정**이다. Claude는 자기 토큰 사용량을 실시간으로 읽지 못하므로, 정확한 소모·잔여 한도 확인은 환경별로 다르다 — Claude Code는 `/cost`와 context 게이지, claude.ai 웹/앱은 사용량 안내를 참고한다. 여기 값을 확정 수치처럼 사용자에게 제시하지 않는다.

## 모드별 대략 소모 가늠표

상대 등급만 본다(적음 < 중간 < 많음). 실제 값은 대화 길이·조사량에 따라 크게 달라진다.

| 작업 | 상대 소모 | 무엇이 토큰을 먹나 | 절약 팁 |
|------|-----------|--------------------|---------|
| 빠른 초안 (`bizplan-quick`) | 적음~중간 | 한 번의 전체 생성 | 이 자체가 절약형. 수치는 예시로 채우고 나중에 보강 |
| 특정 섹션 1개 (`bizplan-problem` 등) | 중간 | 인터뷰 왕복 + 생성 | 질문을 한 번에 묶어 왕복을 줄인다 |
| 꼼꼼 전체 파이프라인 | 많음 | **긴 context 누적** + 웹 실조사 | 파트별로 대화를 끊는다 / 조사는 뒤로 미룬다 |
| 시장 조사 (`bizplan-research`) | 중간~많음 | 웹 검색·다수 출처 대조 | 핵심 수치 2~3개만 우선 조사 |
| 심사위원 검토 (`bizplan-review`) | 중간~많음 | 출처 검증 웹 조사 | 검증은 제출 직전 한 번만 |
| 시각자료 (`bizplan-visual`) | 중간 | matplotlib 코드 재실행 반복 | 도식 안을 먼저 확정한 뒤 한 번에 생성 |
| 벤치마킹 분석 (`bizplan-analyze`) | 많음 | 다수 문서(PDF) 통독 + 교차 비교 | 문서를 2~3건으로 추리고 핵심 섹션 위주로 발췌 분석 |
| 슬라이드 제작 (`bizplan-slides`) | 중간~많음 | HTML 생성 + 레이아웃 검증 반복 | 구성안(bizplan-pitch)을 먼저 확정해 재작업을 줄인다 |

## 단계별 모델 추천

모델 변경은 환경별로 다르다(Claude Code: `/model`, claude.ai 웹/앱: 입력창의 모델 선택 메뉴). 스킬은 "권장"만 한다. 원칙: **판단·설득·논리 정합성이 점수를 가르는 단계는 상위 모델, 정리·형식화는 균형 모델, 기계적 다듬기는 경량 모델.**

| 단계·작업 | 권장 모델 | 이유 |
|-----------|-----------|------|
| 문제 프레이밍(`bizplan-problem`), 성장전략(`bizplan-growth`), 심사 검토(`bizplan-review`), 발표 Q&A(`bizplan-pitch`) | **Opus 4.8** (`claude-opus-4-8`) | 논리 정합성·설득력·심사위원 관점 판단이 당락을 가른다 |
| 시장 조사 정리(`bizplan-research`), 실현가능성·일정(`bizplan-solution`), 팀(`bizplan-team`), 개요(`bizplan-summary`) | **Sonnet 5** (`claude-sonnet-5`) | 구조화·정리 중심이라 균형 모델로 충분(속도·비용 이점) |
| 단순 리라이트, 표 형식 정리, 오탈자·문장 다듬기 | **Haiku 4.5** (`claude-haiku-4-5`) | 기계적 작업은 경량 모델이 빠르고 저렴 |

- **상대 비용(대략): Haiku 4.5 < Sonnet 5 < Opus 4.8 < Fable 5.** 구독 요금제에선 "달러"가 아니라 **사용량 한도 소모**로 이해하면 된다.
- **Fable 5(`claude-fable-5`)는 최상위 난도·초장기 작업 전용**이라 사업계획서 작성에는 과하다 — 기본 후보가 아니다.
- **Pro 요금제에서도 Opus 4.8로 전체 파이프라인 진행이 대개 가능하다** — 요금제 이름만 보고 모델을 낮추지 않는다. **Sonnet 5**는 절약이 필요할 때(사용자 요청 또는 한도 도달) 정리·형식화 단계에 쓰는 선택지이지, Pro의 기본값이 아니다. 한도가 빠듯하면 결정적 단계(검토·문제 프레이밍)에만 **Opus 4.8**을 아껴 쓰고 나머지를 **Sonnet 5**로 돌린다.
- **주의:** 요금제(Pro/Max)별 모델 접근·사용 한도는 서로 다르고 바뀔 수 있다. 특정 한도 수치를 단정하지 말고, 사용자의 플랜과 Claude Code 표시를 기준으로 안내한다.
