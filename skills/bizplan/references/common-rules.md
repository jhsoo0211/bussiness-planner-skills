# 공통 규칙: 지원사업 성격 파악 (전 모드 공용)

모든 bizplan 모드는 작업 시작 전 아래를 확인한다. 이 파일이 단일 기준이며, 각 서브스킬에는 한 줄 요약만 둔다. 정책·수치가 바뀌면 **이 파일만 수정**하면 된다.

## 파악할 항목

1. **주관기관과 사업명**: 예비창업패키지 / 청년창업사관학교(청창사) / U300 / 로컬크리에이터 / 수출바우처 / 기타
2. **평가 기준·배점**: 공식 배점표가 있으면 반드시 첨부를 요청한다 — 배점 항목에 본문을 매핑하는 것이 점수의 지름길이다.
3. **지원 규모·기간**: 금액·기간은 해당 공고 기준으로 조정한다. 원본의 예창패 기준 수치(1단계 2천만 + 2단계 4천만, 8개월)는 기본값 예시로만 쓴다.
4. **양식 구조**: 예창패 PSST형 / 청창사 기술성·시장성·사업성형 / U300형 / 로컬크리에이터 지역성형
5. **심사 방식**: 서면 / 발표. 발표 심사가 있으면 초안 완성 후 `bizplan-pitch`로 연결한다.

## 정부지원사업이 아닌 경우

투자 유치 IR, 민간 공모전, 정책자금·대출 심사, 사내벤처, B2B 제안 등이면 해당 심사·평가 자료(IR 가이드라인, 공모 요강, 심사 항목표) 첨부를 요청한다. 자료가 없으면 유형별 부각 포인트를 먼저 안내한다:

- **IR** = 팀·시장·트랙션·유닛 이코노믹스·투자금 사용처(라운드·희망 밸류·엑싯 전략 포함)
- **공모전·경진대회** = 주최 취지 부합·창의성·발표 전달력 가중(재무·양산 계획 비중↓, 통상 5~10페이지 — **공공 주최는 지식재산 귀속·공공 활용 동의 조항이 있는 경우가 많으니 확인하도록 고지**)
- **정책자금·대출** = 상환 능력·재무 건전성
- **B2B 제안** = 고객사 문제 해결과 ROI

이후 각 모드의 출력 형식을 그 유형에 맞게 조정한다.

## 확인 방법

사용자가 모르면 공고문 파일을 요청하거나 웹 검색으로 파악한다. 대화에 이미 정보가 있으면 다시 묻지 않는다.

## U300 등 학생 트랙 — 깊이 인터뷰

학생 팀은 원가 데이터·시제품·매출 실적 같은 증빙 자산이 없는 경우가 많다. 예창패 수준의 깊이를 일괄 적용하지 말고, 아래 3~4가지로 **무엇을 갖고 있는지 항목별로** 먼저 확인한다.

① **원가·단가 데이터** — 견적서·BOM·재료비 계산이 있는가
② **시제품·실측·증빙** — 시제품(사진), 성능 실측치, 시험 결과, 1차 조사(설문·인터뷰)가 있는가
③ **매출·계약 실적** — 판매 이력, 구매의향서(LOI), PoC 협의가 있는가
④ (재무 추정을 쓸 계획이면) **추정 근거** — 고객 수·단가·전환율 가정을 세워 봤는가

**있는 항목은 예창패 수준 깊이로 쓰고, 없는 항목은 "산정 예정(근거·확보 시점 명시)" 폴백으로 처리한다**(`bizplan-solution` B-1 원칙). 전체를 일괄 간략화하지 않는다 — 항목별 판단이 원칙이다. 이 확인 결과는 이후 모드(`bizplan-solution`·`bizplan-research`·`bizplan-review`)가 재사용한다.

## 출력 형식 (세션 공통)

세션에서 정한 형식(채팅/.md/.docx)을 모든 모드가 따른다. 파일 모드면 결과를 파일에 누적하고 채팅에는 요약만 보여준다. 정해진 것이 없으면 채팅 출력을 기본으로 하되, 긴 문서면 파일 저장을 한 번 제안한다. 상세 절차(환경별 파일 처리·.docx 변환)는 `bizplan` SKILL.md의 '출력 형식 (세션 공통)' 절을 참조한다.
