# 사업계획서 시각자료 가이드 (실전 사례 분석 기반)

실제 정부지원사업 사업계획서·공공 제안서의 시각자료를 전수 분석한 결과를 근거로 작성한 처방적 가이드다.
분석 대상: 해조류 업사이클링 화장품 사례(15p, 실적형), 투수 보도블록 B2G 사례(8p, PSST 구조), 바이오에탄올 사례(15p), 교육키트 사례(7p), 공공 감리 제안서 사례(10p, 슬라이드형).
모든 권고는 사례의 페이지·수치를 인용한다. 근거 없는 일반론은 싣지 않았다.

## 잘 쓴 패턴

1. **주장-증거 병치 레이아웃** — 텍스트 주장 바로 옆에 물리적 증거 이미지를 배치한다. 해조류 화장품 사례는 포지셔닝맵 옆에 실제 소비자 리뷰 캡처(p.10), '재생산 요청 쇄도' 옆에 고객 문의 캡처(p.4·8), 글로벌 계획 옆에 해외 DM·입점 제안서(p.14)를 붙였고, 투수 보도블록 사례는 구매의향 표 옆에 건설자재 유통사의 의향서 실물 스캔(p.5)을 병치했다. 심사위원이 "근거는?"을 물을 틈을 시각적으로 차단한다.
2. **톱다운×바텀업 교차 검증 시장규모** — 투수 보도블록 사례(p.3~4)는 동심원(1.2조→2,860억→273.7억, 각 층위 CAGR·출처 병기)에 산정식(투수 기준면적 119만㎡×증가율 8.0%×단가)을 붙이고, 첫해 점유율 0.16%를 실제 확보한 구매의향서 면적(990~1,320㎡)으로 역산했다. 시장 숫자가 '조사'가 아니라 '계약 전 단계 증거'에 닿는다.
3. **모든 그래프에 결론 한 줄 캡션** — 투수 보도블록 사례는 그래프마다 "기존 제품 대비 8배 우수한 투수 성능", "기존 제품 대비 12MPa 높은 강도", "서울시 25개 자치구청 보도관리팀 인터뷰 결과"를 달았다(p.1, p.3). 심사위원이 그래프를 해석하지 않고 캡션만 읽어도 주장이 전달된다.
4. **실적의 원 단위 전면 공개** — 해조류 화장품 사례(p.14)는 채널별 판매금액을 3,584,824원처럼 원 단위까지 단가·시기와 함께 공개했다. 총액(약 2,800만원)이 크지 않아도 정밀한 투명성이 오히려 신뢰를 만든다.
5. **상태 배지가 달린 단계 도식의 일관 재사용** — 투수 보도블록 사례는 검증 로드맵 4단계(완료/진행 예정/PoC 적용/인증 추진, p.3)와 시장 진입 3단계(p.6)에 같은 시각 문법(번호 카드+검증기관+상태 배지)을 썼다. 완료와 계획의 정직한 구분이 신뢰를 높이고 문서에 디자인 일관성을 준다.
6. **원가 분해로 가격 경쟁력 논증** — 투수 보도블록 사례(p.4)는 판매가를 원재료비+운송비+생산수수료로 쪼개고 연도별 조치(위탁→양산→자체 분쇄)가 어느 항목을 줄이는지 보여, 판매가 인하(28,750→23,000원)와 마진 상승(20%→51.1%)이 동시에 성립하는 이유를 증명했다. 해조류 사례(p.9)도 MOQ·원가율(37%→27%)을 카드로 공개했다.
7. **발로 뛴 1차 조사의 현장 증빙** — 해조류 화장품 사례(p.3~4)는 어촌계장·해녀 회장·선장 실명 인터뷰 사진 6장과 도청 미팅("현재로서는 처리 불가능" 인용)을 배치했고, 투수 보도블록 사례(p.1)는 25개 자치구 인터뷰를 파이차트로 정량화했다. 데스크 리서치와 격이 다른 설득력을 만든다.
8. **정량화 + 산정식 각주** — 해조류 화장품 사례(p.6)는 "CO2 1,860kg 절감"에 산정식(제품 1개당 304g, 해조류 1kg 소각 시 2.2kg 절감)을 각주로 붙이고 과거 실적(500kg)→협약기간 목표(+1t, 누적 4,668kg)로 연장했다. 임팩트 지표를 검증 가능한 형태로 만드는 모범이다.

## 피해야 할 패턴

1. **빈 표·이미지로만 삽입된 표** — 교육키트 사례는 핵심 표 9개를 전부 이미지로 삽입해, 텍스트 추출 시 캡션만 남고 내용이 사라져 "표가 비어 있다"는 오독을 유발했다(검색·복사·정량 검증·접근성 모두 불가). 공공 감리 제안서 사례는 승부처인 유사 실적 영역(p.10)이 내용 없는 빈 카드로 잘린 채 제출됐다. **표는 반드시 텍스트 기반으로 작성하고, 제출 전 모든 표에 내용이 채워졌는지 최종 렌더를 검수하라.**
2. **숫자 검산이 안 맞는 표** — 해조류 사례: 매출을 p.1 '2500(백만원)'=25억, 본문 '2,500만원', p.14 성과표 '25,000만원'=2.5억으로 3중 모순(100배·10배 차이). 투수 보도블록 사례 p.7: 자금표 개별 합 2억 800만원 vs 합계 2억 8,000만원. 교육키트 사례 표5: 1.5조×0.1%=15억을 150억으로, 29조×0.01%=29억을 290억으로 10배 과다. 바이오에탄올 사례 p.7: "연평균 67% 성장"이 550억→900억 달러(실제 약 5% CAGR)와 모순. 심사위원의 암산 한 번에 문서 전체 신뢰가 무너진다.
3. **출처 없는 핵심 수치** — 해조류 사례 p.12의 TAM/SAM/SOM(22.8조·13.8조·7.8조·9,336억)과 비율(40%·90.7%·95.3%) 전부 출처 미표기, p.7 "히알루론산 대비 130% 보습력"도 시험기관 미표기. 같은 문서에서 출처를 단 그래프(조선일보, Statista·KOFICE)와 대비되어 미표기 수치의 신뢰가 더 깎였다. 교육키트 사례의 SOM 810억도 산정근거 부재.
4. **판독 불가 수준의 축소 캡처** — 해조류 사례 p.7 효능 그래프(핵심 차별화 근거인데 축·수치 안 보임), 바이오에탄올 사례 p.10 원가우위 61.2%의 유일한 근거인 논문 표(셀 판독 불가), 투수 보도블록 사례 p.6 규제샌드박스 재촬영 캡처. 핵심 그래프는 재작도하고 원본은 별첨하라.
5. **동일 시각자료의 중복 배치** — 해조류 사례는 스케일업 타임라인(p.2=p.8), 공정도(p.4=p.9), 문의 캡처(p.4=p.8)가 2회씩 등장해 15p 중 1p 이상을 낭비했다. 요약과 본문은 정보 깊이를 다르게 하거나 참조 표기로 대체하라.
6. **근거 없는 점유율 도약** — 투수 보도블록 사례 p.3: 첫해 0.16%는 구매의향서로 치밀하게 뒷받침했으나 2028년 15%→2029년 35.6%는 근거가 전혀 없어, 앞의 치밀함과 대비되어 오히려 눈에 띄었다.
7. **강조 남발** — 투수 보도블록 사례 p.1~4는 형광펜·빨강·밑줄·굵기가 한 문단에 혼용되어 페이지의 30% 이상이 강조 상태. 강조가 많으면 우선순위가 사라져 핵심 수치가 묻힌다. 강조는 페이지당 1~2곳으로 제한하라.
8. **정량 지표 없는 정성 서술** — 공공 감리 제안서 사례는 그래프·차트가 0개, 기대효과 8개 항목이 전부 '단축·최소화·절감'식 서술(p.4·9)이었고, 교육키트 사례 경쟁표(표4)도 전 항목이 '높음/보통' 정성 평가였다. 목표 수치가 없으면 기억에 남는 숫자도 없다.
9. **시점·조건이 다른 수치의 비교** — 투수 보도블록 사례 p.5: 자사 단가 23,000원은 2028년 목표가인데 경쟁 3사의 현재 가격과 나란히 비교(2026년 실판매가는 28,750원으로 더 비쌈). 비교표의 모든 수치는 동일 시점·동일 조건이어야 하며 측정 조건을 각주로 밝혀라.

## 시각자료 유형별 제작 절차

### 1) 시장규모 그래프 (막대 + CAGR + 추세선)
- **데이터 준비**: 최근 4~5개년 시장규모를 공신력 있는 출처(KMAPS, 통계청, 업계 보고서)에서 확보. 통화·단위(억원/조원) 통일.
- **산정/계산**: CAGR = (말년값/초년값)^(1/연수) − 1. 계산 결과를 보고서 인용치와 교차 검증(바이오에탄올 사례의 "67%" 같은 모순 방지).
- **도식화**: 연도별 막대 + 점선 추세선 + CAGR 라벨. 막대 위에 수치 라벨, 하단에 출처 각주. 결론 한 줄 캡션(예: "연 6.6% 성장하는 2,860억 시장").
- **검증**: 축 단위 표기 확인(투수 보도블록 사례의 '6.0E-02' 과학표기+단위 누락 반면교사), 출처·기준연도 병기 확인.

### 2) 비교표 (O/△/X vs 실측치)
- **데이터 준비**: 경쟁 2~3사 선정, 비교 항목 4~6개(고객의 구매 판단 요소에서 도출 — 투수 보도블록 사례는 세그먼트별 구매 기준 조사 결과를 항목으로 씀).
- **산정/계산**: O/△/X 판정 기준을 먼저 정의하고, 가능한 항목은 실측치를 병기(휨강도 △(4.0) vs O(5.0)처럼). 전 항목 자사 O는 자의성 의심을 산다(해조류 사례 p.10의 약점).
- **도식화**: 자사 행/열을 색 테두리·배경으로 강조(바이오에탄올 사례의 녹색 행, 교육키트 사례의 주황 테두리), 범례("O: 완전 가능…") 필수.
- **검증**: 모든 수치의 시점·측정 조건 통일 여부, 경쟁사 수치 출처(조달청 공시가/실측) 각주 여부.

### 3) 포지셔닝맵 (2축)
- **데이터 준비**: 고객 구매 판단 요소 중 자사가 우위이면서 **고객이 실제로 중요시하는** 2개 축 선정. 축 선정 근거를 본문에 한 줄로 명시(예: "자치구 인터뷰에서 구매 기준 1·2위가 투수 지속성과 단가").
- **산정/계산**: 각 사 좌표의 근거 데이터 확보(자사=실측/리뷰, 경쟁사=공개 스펙·리뷰). 해조류 사례 p.10은 자사 좌표를 리뷰 캡처·구매 544명과 연결해 '주장+증거' 쌍으로 만들었다.
- **도식화**: 자사를 우상단(목표 포지션)에 색 강조 배치, 경쟁사는 로고로 표기. 축 끝에 방향 라벨.
- **검증**: 축에 눈금·수치가 없으면 정성 주장에 그친다(바이오에탄올 사례 그림8의 한계). 최소한 경쟁사 좌표의 근거를 각주로.

### 4) BM 캔버스 / 구성도
- **데이터 준비**: 이해관계자 목록(공급자·생산·고객·회수), 각 관계에서 오가는 것(돈/물자/데이터)을 정리. 파트너는 실명으로(투수 보도블록 사례 p.8: 협력기관 8곳 실명+시기).
- **산정/계산**: 수익원별 단가·마진 숫자를 준비해 화살표 옆에 붙일 수 있게.
- **도식화**: 순환·양방향 구조가 핵심이면 커스텀 구성도(투수 보도블록 사례 p.4의 원료 무상 수급↔판매↔회수 순환)가 BM 캔버스보다 낫다. 표준 9블록 캔버스는 사업 전모 조망용이나 텍스트 과밀 주의(교육키트 사례 표7은 10초 파악 실패).
- **검증**: 도식의 각 노드에 실존 파트너가 대응되는지(BM 도식과 협력기관 표의 상호 참조).

### 5) 간트 / 로드맵
- **데이터 준비**: 과업 목록을 트랙(제품/사업/인증 등)으로 묶고 월 단위 기간, 마일스톤(런칭·인증·PoC), 정량 목표를 정리.
- **산정/계산**: 매출 목표·원가 계획과 연도별 정합성 확인(투수 보도블록 사례는 3개년 로드맵이 p.3 매출 목표·p.4 원가표와 정확히 맞물림).
- **도식화**: 트랙별 색 구분(해조류 사례 p.9), 마일스톤은 다이아몬드+라벨, 완료/진행/예정 상태 배지. 확정 전 일정은 "* 협의 후 확정" 각주(공공 감리 제안서 사례 p.8의 좋은 디테일).
- **검증**: 빈 바·라벨 누락 렌더링 오류 검수(공공 감리 제안서 사례 p.8의 빈 흰색 바 반면교사).

### 6) 문제-솔루션 1:1 대응 도식
- **데이터 준비**: 문제를 2~3개로 수렴시키고 각 문제의 정량 지표(규모·비용·추이)를 확보.
- **산정/계산**: 각 솔루션 요소가 해결하는 문제와 개선 폭을 수치화(투수 보도블록 사례 p.2: "50배 압축 강도", "공극 +10%").
- **도식화**: 번호 매긴 솔루션 카드 병렬 배치, 카드마다 고유 시각 형식(미니 그래프/단면도/공정사진)과 핵심 수치 1개. 해조류 사례는 문제 1(해양폐기물)+문제 2(해녀 소멸)가 '해녀와 함께 수거'라는 하나의 솔루션으로 수렴하는 복선 구조(p.3~4)를 썼다.
- **검증**: 문제 없이 등장하는 솔루션, 솔루션 없이 방치된 문제가 없는지 1:1 대응 확인.

## TAM/SAM/SOM 심화

### 산정 방법론: 하향식 vs 상향식
- **하향식(Top-down)**: 시장보고서·정부 통계의 전체 시장에서 지리·행태·인구 필터로 좁혀 내려온다. TAM·SAM처럼 큰 층위에 적합. 해조류 화장품 사례(p.12)의 '글로벌 22.8조→국내 13.8조→컨셔스뷰티 40%→3040 여성 70%' 필터 체인이 전형(단, 출처 부재로 실패).
- **상향식(Bottom-up)**: 고객 수 × 단가 × 전환율로 쌓아 올린다. SOM·첫해 매출처럼 자사가 실제로 잡을 숫자에 적합. 투수 보도블록 사례(p.4)는 첫해 점유율 0.16%(매출 4,500만원)를 '확보한 구매의향서 면적 990~1,320㎡ + PoC 희망 지자체 소규모 시공 면적 × 제품 단가'로 역산했다.
- **원칙**: TAM/SAM은 하향식, SOM은 상향식으로 산정한 뒤 두 값이 만나는지 교차 검증하라. 하향식만 쓰면 "큰 시장의 몇 %" 논리가 공중에 뜨고, 상향식만 쓰면 시장의 크기를 못 보여준다.

### 근거 출처 유형
| 층위 | 적합한 출처 | 사례 |
|---|---|---|
| TAM | 통계청·정부 기본통계, 글로벌 시장보고서(Statista 등) | 투수 보도블록 사례: KMAPS 기준 1.2조 |
| SAM | 정부 계획·조례, 업계 보고서, 산업 협회 통계 | 투수 보도블록 사례: 서울시 조례 제9812호(2027년 투수 1급 의무화)가 시장 기회의 규제적 근거 |
| SOM | 자체 조사 — 구매의향서, 담당자 인터뷰, PoC 협의 기록 | 투수 보도블록 사례: 구매의향서 면적 990~1,320㎡, 25개 자치구 인터뷰 |

### 실전 사례 해부: 모범 vs 실패
**모범 — 투수 보도블록 B2G 사례(p.3~4)**
- 전체(TAM) = 국내 건설용 블록 시장 1.2조원(CAGR 5.3%): 옹벽·경계석·콘크리트 제품 전체.
- 유효(SAM) = 전국 보도블록 시장 2,860억원(CAGR 6.6%): 투수·불투수 포함.
- 목표(SOM 모집단) = 서울시 투수블록 시장 273.7억원(CAGR 8.0%) = **서울시 투수 기준면적 119만㎡ × 연평균 증가율 8.0% × 제품 단가** — 산정식을 도식 아래 본문에 명시. 출처(KMAPS, 서울시 도로관리과 2025) 병기.
- 진출년도 점유율 0.16%(매출 4,500만원)는 **구매의향서(990~1,320㎡)와 PoC 시공 면적**이라는 종이 증거로 상향식 뒷받침. 톱다운 시장규모와 바텀업 초기 점유율의 교차 검증이 이 문서의 최대 강점.
- 남은 약점까지 배우라: 2028년 15%→2029년 35.6% 점유율은 근거가 없어 0.16%의 치밀함과 대비되어 오히려 신뢰를 깎았다. **모든 연도의 점유율에 산정 로직을 달거나, 근거가 없으면 보수적으로 쓰라.**

**실패 — TAM만 나열한 교육키트 사례(표5)**: 사교육 29조·과학교구 1.5조라는 큰 숫자에서 멈추고 SOM 810억의 산정근거가 없다. 게다가 시장규모×점유율=추정매출 산식이 10배씩 틀렸고(1.5조×0.1%=15억을 150억으로), 29조의 정의가 문서 내에서 '사교육 시장'과 '과학 교구 키트 전체 시장'으로 오락가락했다. 바이오에탄올 사례도 같은 실패형: 전 세계 550억 달러 TAM만 던지고 국내 SAM·목표 SOM 정량화가 전무해 "큰 시장의 몇 %" 논리가 성립하지 않았다(p.7).

### 시각화 형식: 동심원 vs 깔때기 vs 표
- **동심원(권장 기본형)**: 포함 관계(TAM⊃SAM⊃SOM)를 직관 전달. 투수 보도블록 사례처럼 각 원에 금액+CAGR, 도식 밖에 산정식·출처.
- **깔때기/계단식**: 필터 체인(지리→행태→인구)을 강조할 때. 해조류 사례 p.12처럼 우측에 근거 텍스트 패널을 분리 배치하는 형식 자체는 10초 파악이 가능해 우수했다(출처 부재가 문제였을 뿐).
- **표/스탯타일**: 층위별로 '시장규모·목표점유율·추정매출' 3지표를 비교할 때. 교육키트 사례 표5의 카드 형식은 시각적으로 우수했으나 산술 오류로 무너졌다.
- **공통 원칙: 어떤 형식이든 각 층위에 금액 + 산정근거 한 줄을 반드시 병기하라.** 금액만 있는 층위는 심사에서 반박당한다.

### 흔한 실수 체크
- 근거 없는 점유율 가정(투수 보도블록 사례 35.6%, 교육키트 사례 0.1%·0.01% 임의 설정)
- TAM만 크게 쓰고 SOM 산정 로직 부재(교육키트·바이오에탄올 사례)
- 층위 정의 불일치·산식 불연결(교육키트 사례 29조 라벨 모순, 해조류 사례 13.8조×40%=5.5조≠7.8조)
- 층위 간 곱셈 검산 생략(교육키트 사례의 10배 오류) — 제출 전 계산기로 전 층위 검산

### matplotlib 동심원 TAM/SAM/SOM 코드

```python
import platform
import matplotlib.pyplot as plt
from matplotlib.patches import Circle

if platform.system() == "Windows":
    plt.rcParams["font.family"] = "Malgun Gothic"
elif platform.system() == "Darwin":
    plt.rcParams["font.family"] = "AppleGothic"
else:
    plt.rcParams["font.family"] = "NanumGothic"
plt.rcParams["axes.unicode_minus"] = False

# 예시 데이터(투수 보도블록 B2G 사례 구조) — 실제 작성 시 금액·근거·출처 교체
layers = [  # (이름, 시장 정의, 금액·CAGR, 산정근거 한 줄, 반지름, 색)
    ("TAM", "국내 건설용 블록 시장", "1.2조원 · CAGR 5.3%",
     "옹벽·경계석 등 건설용 블록 전체", 1.00, "#D6E4F0"),
    ("SAM", "전국 보도블록 시장", "2,860억원 · CAGR 6.6%",
     "투수·불투수 포함 보도블록", 0.62, "#8FB8DE"),
    ("SOM", "서울시 투수블록 시장", "273.7억원 · CAGR 8.0%",
     "투수 기준면적 119만㎡ × 증가율 8.0% × 단가", 0.32, "#2E6DA4"),
]

fig, ax = plt.subplots(figsize=(10, 7))
ax.set_xlim(-1.15, 2.8)
ax.set_ylim(-1.2, 1.15)
ax.set_aspect("equal")
ax.axis("off")

for i, (name, label, value, basis, r, color) in enumerate(layers):
    ax.add_patch(Circle((0, r - 1.0), r, facecolor=color,
                        edgecolor="white", linewidth=2.5, zorder=i + 1))
    top_y = 2 * r - 1.0                      # 각 원의 꼭대기(아래 끝 y=-1 정렬)
    ax.text(0, top_y - 0.12, name, ha="center", fontsize=13, fontweight="bold",
            color="white" if i == 2 else "#1A2A3A", zorder=10)
    # 우측 리더라인: 금액 + 산정근거 한 줄 병기(핵심 원칙)
    ax.annotate(f"{name}  {label}\n{value}\n근거: {basis}",
                xy=(0.18, top_y - 0.16), xytext=(1.30, top_y - 0.10),
                fontsize=9.5, va="top",
                arrowprops=dict(arrowstyle="-", color="#666666", lw=1))

ax.text(0.8, -1.15, "출처: KMAPS·서울시 도로관리과(2025) — 실제 출처로 교체",
        fontsize=8, color="gray", ha="center")
plt.tight_layout()
plt.savefig("tam_sam_som.png", dpi=200, bbox_inches="tight")
```

### matplotlib (한글 폰트 설정 포함)

```python
import platform
import numpy as np
import matplotlib.pyplot as plt

if platform.system() == "Windows":
    plt.rcParams["font.family"] = "Malgun Gothic"
elif platform.system() == "Darwin":
    plt.rcParams["font.family"] = "AppleGothic"
else:
    plt.rcParams["font.family"] = "NanumGothic"
plt.rcParams["axes.unicode_minus"] = False

# 예시 데이터: 시장규모 추이(억원) — 실제 작성 시 수치·출처 교체
years = [2021, 2022, 2023, 2024, 2025]
size = [2210, 2355, 2510, 2675, 2860]
cagr = (size[-1] / size[0]) ** (1 / (len(years) - 1)) - 1  # CAGR 산식

fig, ax = plt.subplots(figsize=(8, 5))
bars = ax.bar(years, size, color="#4C72B0", width=0.6)
z = np.polyfit(years, size, 1)                             # 추세선
ax.plot(years, np.poly1d(z)(years), "--", color="#C44E52",
        linewidth=2, label=f"추세선 (CAGR {cagr * 100:.1f}%)")
for b, v in zip(bars, size):                               # 막대 위 수치 라벨
    ax.text(b.get_x() + b.get_width() / 2, v + 25, f"{v:,}",
            ha="center", fontsize=10)

ax.set_title("국내 보도블록 시장 규모 추이", fontsize=14, pad=12)
ax.set_ylabel("시장 규모 (억원)")                            # 축 단위 필수 표기
ax.legend(loc="upper left")
ax.spines[["top", "right"]].set_visible(False)
fig.text(0.99, 0.01, "출처: KMAPS(2025) — 실제 출처로 교체",
         ha="right", fontsize=8, color="gray")
plt.tight_layout()
plt.savefig("market_size.png", dpi=200)
```

포인트: 축 단위를 반드시 라벨에 쓰고(투수 보도블록 사례의 '6.0E-02' 무단위 반면교사), 출처를 그림 안에 각주로 박아 그림만 복사돼도 출처가 따라가게 하라.

### 이미지 생성 AI (GPT-4o·Gemini·Nano Banana 등) — 개념 도식용 영문 프롬프트

이미지 생성 AI는 **개념 도식·일러스트(서비스 개요도, 사용 시나리오 컷, BM 순환 구성도)**에만 쓴다. 수치·축이 들어가는 그래프는 숫자를 정확히 그리지 못하므로 matplotlib로 만든다(위 절 참조). 편집 가능한 구조 도식이 필요하면 냅킨AI가 낫다.

**원칙**

1. 프롬프트는 영어로 쓴다(이미지 모델은 영어 지시 추종이 더 정확하다). 단 도식 안 텍스트·라벨은 전부 한국어로, 프롬프트에 따옴표로 그대로 지정하고 "번역·변형 금지"를 명시한다.
2. 스타일은 정부지원사업 문서 톤으로 고정한다: 플랫·미니멀 벡터, 흰 배경, 기본 1색+강조 1색(아래 PPT 도식 팁과 동일 원칙). 미국식 클립아트 인물·핸드셰이크 스톡 사진·만화 마스코트·달러 기호를 금지 항목으로 프롬프트에 적는다.
3. 생성 후 한국어 라벨을 지정 문구와 전수 대조한다. 이미지 모델은 한글 획을 자주 뭉갠다 — 깨진 라벨은 재생성하거나, 라벨 없이 생성한 뒤 PPT·이미지 편집 도구로 텍스트를 얹는다.
