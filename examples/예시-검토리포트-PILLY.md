## 2. 강점 (3가지)

1. **문제-솔루션 완전 1:1 대응(부록 A).** P1·P2·P3 ↔ S1·S2·S3 고아 문제·고아 솔루션 0건. 복약 데이터 분석을 "부가 편익"으로 명확히 격하해 고아 솔루션화를 스스로 차단 — 심사위원에게 "정리된 팀" 인상을 준다.
2. **분야 필수 게이트를 선제 인지(2-4).** 복약 데이터의 의료기기(SaMD) 해당 가능성을 인지하고 "비의료기기 포지션 → 식약처 사전상담 → 재검토" 단계 전략을 세웠다. KC·전기안전도 로드맵에 배치. case-insights의 "규제 무지형" 감점을 회피했다.
3. **TAM 과시를 피하고 SOM을 상향식·보수적으로 산정.** SAM의 약 0.5%(ARR 43억)만 목표로 잡고, 일시성 HW 매출을 ARR에서 분리했다. 검산도 내적으로 일치(8번 참조) — case-insights의 "TAM 과시형"·"숫자 모순형" 실패를 동시에 회피.

## 4. 문제-솔루션 1:1 대응 점검

| 문제 (P)                                          | 대응 솔루션 요소 (S)                              | 상태                                                                            |
| ------------------------------------------------- | ------------------------------------------------- | ------------------------------------------------------------------------------- |
| **P1** 디지털 소외(폰 못 씀)                | **S1** 폰 없이 쓰는 오프라인 스마트 약통    | **대응됨**                                                                |
| **P2** 복용 여부 확인 불가(자가보고 부정확) | **S2** 복용 감지 센서(실측 데이터)          | **대응됨** (단, S2 감지 정확도 미검증 → 대응의 *실현성*은 별도 리스크) |
| **P3** 이상징후 대응 지연                   | **S3** 미복용 실시간 알림 + 지자체 대시보드 | **대응됨**                                                                |
| (참고) 복약 데이터 분석·건강 조기징후            | 부가 편익(초기 범위 밖으로 격하)                  | **고아 솔루션 아님** (편익으로 명시 격하)                                 |

- **판정:** 고아 문제 **0건**, 고아 솔루션 **0건**. 대응 구조 자체는 모범. **단 P2↔S2는 "논리적 대응"은 완성됐으나 감지 정확도 실증이 없어 "실현적 대응"이 미완** — 이 점을 발표에서 반드시 방어해야 한다.

## 6. 필수 수정 사항

1. **원가-마진-BEP 표 신설**(성장전략) — 대당 제조원가·SaaS 한계이익·손익분기 대수를 3개년으로. *현재 가장 큰 공백.*
2. **개인정보(민감 건강정보) 처리·보안 방안 + 영향평가 계획 신설**(실현가능성/규제) — B2G 필수 관문.
3. **경쟁 분석 심화** — 카테고리 서술을 개별 제품 가격·기능 **실측 비교표**로, 포지셔닝맵 좌표에 근거 각주.
4. **핵심 [예시 추정] 정박** — TAM 20%·단가 40만·전환 60%에 3자 근거 최소 1개씩.
5. **실증 앵커 전면화** — PoC MOU·LOI·정확도 벤치테스트 수치를 로드맵 최우선 배치, 실물 산출물 캡처/URL 노출, "확보/목표" 구분.

## 7. 발표 시 예상 질문

> **유도 질문(우리가 원하는):** 강점을 자연스럽게 부각하게 하는 질문. 답변을 준비해 대화를 이 방향으로 끌고 간다.
> **위험 질문(약점 방어):** 심사위원이 약점을 찌르는 질문. 방어 논리를 미리 확보한다.

**유도 질문 (우리가 원하는)**

1. "폰 없이 작동한다는 게 정확히 어떻게 되나요?" → S1 오프라인 LTE-M 내장·음성/불빛/큰버튼 UX 설명 기회.
2. "왜 지금·왜 이 사업인가요?" → 초고령사회 + **정책 공백(응급안전안심·맞춤돌봄이 복약 데이터를 안 다룸)**을 부각(1-4).
3. "SOM을 왜 이렇게 보수적으로(SAM의 0.5%) 잡았나요?" → 상향식 산정·검산·확장은 중장기 KPI라는 신중함 강조.

**위험 질문 (약점 방어)**
4. **[방어 필요]** "대당 원가와 손익분기는요? 월 3만 원 SaaS로 마진이 남나요?" → *현재 방어 불가.* 원가표·BEP 신설이 선행돼야 함(필수 수정 1).
5. **[방어 필요]** "복약 데이터는 민감정보인데 동의·보안·지자체 이관은 어떻게 처리하나요?" → *현재 미검토.* 개인정보 처리방안 신설이 선행(필수 수정 2).
6. **[방어 준비됨]** "매출·계약 실적이 없는데 시장성을 어떻게 증명하나요?" → 지자체 PoC LOI·보호자 사전신청 리드를 앵커로 제시.
7. **[부분 방어]** "MedMinder·Hero와 가격·기능이 구체적으로 뭐가 다른가요?" → 폰 불필요×실측×지자체 연동 3중 차별은 있으나 **가격·스펙 실측 비교표 보강 필요**(필수 수정 3).
8. **[방어 준비됨]** "복용 감지 정확도 95%의 근거는?" → M1–2 벤치테스트·3자 시험 계획으로 방어(수치 확보 후 강화).

**출처 없는 수치의 산출 논리 재구성**

- **TAM "헬스케어 비중 20%" — 왜 20%인가?** 시니어 산업 168조(2030)에 20%를 곱해 33.6조를 얻었다. 그러나 168조는 주거·여가·금융·식품·요양·의약품·의료기기 등을 모두 포함하는 총량이며, **문서 어디에도 "헬스케어=20%"를 뒷받침하는 분해가 없다.** 재구성을 시도하면: 정부 **고령친화산업 8대 분야**(요양·의약품·의료기기·화장품·식품·용품·주거·여가) 중 헬스케어 인접(요양+의약품+의료기기) 비중을 통계로 합산해야 20%의 타당성을 검증할 수 있는데, 문서는 이 과정을 생략했다. → **산출 논리 재구성 실패, 감점.** 8대 분야 통계로 대체 산출을 요구한다.
- **SOM ARR 43억 — 검산.** SaaS = 1만 대 × (월 3만 × 12 = 36만/년) = **36억**. B2C = 1만 가구 × 전환 60% × (월 9,900 × 12 = 11.88만/년) = 6,000 × 11.88만 = **약 7.1억**. 합 = **약 43억** ✓ **검산 통과**(HW 30억은 일시성이라 ARR에서 정확히 분리 — 올바른 처리). SAM 검산도 993.8만 × 23.7% × 90% × 40만 = **약 8,480억** ✓. → **산술적 내적 일관성은 우수**(case-insights "숫자 모순형" 회피). **다만 곱셈에 들어간 전환율 60%·단가가 [예시 추정]이라, 검산은 통과해도 근거는 미검증.** PoC 실측·LOI로 하부 가정을 정박해야 신뢰도가 "보통→높음"으로 올라간다.

## 10. 종합 결론

PILLY 사업계획서는 **구조·논리 정합성(문제-솔루션 1:1, 규제 인지, 보수적 SOM, 검산 일관성)에서 상위권**이나, **정량 증빙(원가·실적)과 개인정보 리스크**라는 세 공백이 발표 단계 실점 요인이다. 현재 72점, 아래 3개를 반영하면 80점대 진입이 현실적이다.

**제출 전 우선순위 액션 3개**

1. **원가-마진-BEP 표 신설** — 성장전략의 최대 공백. 대당 제조원가·SaaS 한계이익·손익분기 대수를 3개년 마진 추이로. (VC 렌즈 정면 대응)
2. **복용 데이터 = 민감 건강정보 처리방안 + 개인정보 영향평가 계획 추가** — B2G 필수 관문. 동의·저장·위탁·파기·지자체 이관 시 처리 주체를 명확화. (공공 담당자 렌즈 정면 대응)
3. **실증 앵커를 로드맵 최우선 배치** — 지자체 PoC MOU·보호자 사전신청 리드·복용 감지 정확도 벤치테스트 수치를 "확보/목표" 구분해 전면 노출. (창업 교수·VC 공통 렌즈 대응)
